# SMAScore 要件定義

## プロジェクト概要

SMAScoreは、**モルック専用**のライブ配信オーバーレイシステムである。

目的は、大会や練習配信で入力しやすく、視聴者にも分かりやすいスコア表示を提供すること。本プロジェクトはモルック競技に特化して設計・開発する。

---

## 対象ユーザー

| ユーザー | 役割 |
|----------|------|
| 配信者 | ライブ配信を行い、オーバーレイを視聴画面に表示する |
| 大会運営者 | 試合進行に合わせてスコアを入力・管理する |
| 視聴者 | オーバーレイを通じて試合状況・スコアを把握する |

---

## 開発方針

- 配信現場で使いやすいことを最優先とする
- 誤操作を防ぐ
- **スマートフォンでの操作を前提**とする
- オーバーレイ表示を重視する
- **OBS・PRISM 配信**での利用を想定する

---

## 設計の基本単位

スコア管理の基本単位は **「チーム」** とする。「プレイヤー」ではなくチーム単位でスコア・表示・試合進行を設計する。

---

## 配信連携

- OBS ブラウザソースとしてオーバーレイ URL を読み込めること
- PRISM Live Studio 等の配信ソフトでも同様にブラウザソースとして利用できること
- 操作画面（スマホ）とオーバーレイ表示は分離し、リアルタイムで同期すること

---

## 機能要件

### 試合設定

試合開始前に、大会・試合の基本情報と形式を設定できること。

#### 試合形式

以下のいずれかを選択できること。

| 形式 | 内容 |
|------|------|
| 勝数先取 | 2～5勝先取 |
| ゲーム合計 | 2～4ゲーム合計 |

#### 対戦チーム数

以下のいずれかを選択できること。

- 2チーム
- 3チーム
- 4チーム

#### その他の設定項目

- チーム名の設定（対戦チーム数に応じた数のチーム名を登録できること）
- 大会名の設定
- 試合名の設定

---

### 試合管理

試合進行中の状態を把握・制御できること。

| 機能 | 要件 |
|------|------|
| 現在セット得点表示 | 各チームの現在セットにおける得点を表示すること |
| 総得点表示 | 各チームの試合全体における総得点を表示すること |
| セットカウント | 各チームの獲得セット数を表示・管理すること |
| 投擲順表示 | 現在の投擲順（どのチームが投擲中か）を表示すること |
| 投擲順自動切替 | 入力確定後、次のチームへ投擲順を自動で切り替えること |
| 入力待ち表示 | 現在入力待ちのチームを明示すること |
| 試合終了判定 | 設定した試合形式に基づき、試合終了を自動判定すること |

---

### モルックルール

モルック競技のルールをシステム上で正しく反映すること。

| ルール | 内容 |
|--------|------|
| 勝利条件 | 50点ちょうどでセット勝利とする |
| オーバーシュート | 50点を超えた場合、得点を25点へ戻す |
| ファール時の減点 | 37点以上の状態でファールした場合、得点を25点へ戻す |
| 失格 | 3回連続ミスで失格とする |
| ファールとミスの区別 | ファール（F）とミス（0）は別々に管理すること |

---

### 入力機能

スマートフォンから、試合進行に合わせてスコアを入力できること。

| 機能 | 要件 |
|------|------|
| 得点入力 | 0～12の得点を入力できること |
| ファール入力 | F（ファール）を入力できること |
| Undo | 直前の入力を取り消せること |
| 修正モード | 過去の入力を修正できるモードを提供すること |
| 現在入力確認 | 確定前に、入力内容を確認できること |
| 決定ボタン | 決定ボタンによる操作で、入力内容を試合状態へ反映すること |
| 入力履歴保存 | 入力の履歴を保存し、後から参照できること |

---

### オーバーレイ

配信画面に表示するオーバーレイとして、視聴者が試合状況を把握できる情報を提供すること。

| 表示項目 | 要件 |
|----------|------|
| 現在得点 | 各チームの現在セットにおける得点を表示すること |
| 総得点 | 各チームの試合全体における総得点を表示すること |
| セット | 各チームの獲得セット数を表示すること |
| ミス表示 | 連続ミス状況を「○○○」形式で表示すること |
| 入力待ち表示 | 現在入力待ちのチームを表示すること |
| 大会名 | 設定した大会名を表示すること |
| 試合名 | 設定した試合名を表示すること |

---

## 操作フロー

SMAScore の基本操作は、試合の作成から投擲の繰り返しまで、以下の流れで行う。

### 基本操作フロー

#### ① 試合を作成

大会名・試合名・試合形式・対戦チーム数など、試合に必要な設定を行う。

#### ② チーム名入力

対戦チーム数に応じて、各チームの名称を入力する。

#### ③ 試合開始

設定内容を確定し、試合を開始する。最初の投擲順が設定され、入力待ち状態となる。

#### ④ 得点入力

現在入力待ちのチームについて、0～12 の得点、または F（ファール）を入力する。

#### ⑤ 入力確認

決定前に、入力した内容が正しいかを確認する。誤りがあれば、入力をやり直す。

#### ⑥ 決定

入力内容に問題がなければ、決定ボタンで確定する。

#### ⑦ 自動処理

決定後、システムが以下の処理を自動で行う。

- 現在得点の更新
- 総得点の更新
- ミス状況の更新
- 投擲順の切替
- オーバーレイの更新
- 入力履歴の保存
- 勝敗判定（セット勝利・試合終了の判定を含む）

#### ⑧ 次の投擲

試合が継続中であれば、次のチームの入力待ち状態へ進む。④～⑧を試合終了まで繰り返す。

---

### 修正モードの操作フロー

試合進行中に過去の入力内容を修正する場合は、通常操作から修正モードへ切り替えて行う。

#### ① 修正モードへ入る

試合進行中の画面から、修正モードを開始する。

#### ② 修正対象を選択

入力履歴から、修正したい投擲を選択する。

#### ③ 入力内容を修正

選択した投擲について、得点（0～12）または F（ファール）を再入力する。

#### ④ 修正内容を確認

決定前に、修正後の内容が正しいかを確認する。

#### ⑤ 修正を決定

修正内容に問題がなければ、決定ボタンで確定する。

#### ⑥ 自動再処理

修正確定後、システムが以下の処理を自動で行う。

- 修正対象以降の試合状態を再計算する
- 現在得点の更新
- 総得点の更新
- ミス状況の更新
- 投擲順の更新
- オーバーレイの更新
- 入力履歴の更新
- 勝敗判定の再実行

#### ⑦ 通常操作へ戻る

修正モードを終了し、現在の試合状態に基づいて通常の入力待ち状態へ戻る。必要に応じて、④～⑧の基本操作フローを継続する。
